# Documentación del Proyecto: Servicio Automatizado de Extracción de Contactos por Correo Electrónico

## 🎯 Propósito del Proyecto

El **"Servicio de Extracción de Contactos por Correo Electrónico"** es una solución automatizada diseñada para **extraer, procesar y almacenar datos de contacto (nombres, cargos, empresas, teléfonos, emails, etc.) directamente desde las firmas de correo electrónico**.

**¿Por qué es necesario?**
En muchas organizaciones, la información de contacto llega a través de emails y a menudo se pierde o requiere entrada manual. Este servicio resuelve ese problema automatizando la captura de datos críticos, mejorando la eficiencia y la calidad de la información de contacto disponible para la empresa.

## 🚀 ¿Cómo Funciona? (Arquitectura y Flujo de Trabajo)

El sistema opera como un **pipeline de procesamiento asíncrono** con una arquitectura de tres capas principales (más una de refinamiento) que trabajan en conjunto:

1.  **Monitoreo IMAP en Tiempo Real:**
    *   El servicio se conecta a una bandeja de entrada de correo electrónico (IMAP).
    *   Está configurado para **monitorear en tiempo real** los correos nuevos o no leídos.
    *   Cada vez que llega un nuevo correo, se activa el proceso de extracción.

2.  **Procesamiento Multi-Capa para la Extracción de Datos:**

    *   **Capa 1: Heurística (Procesamiento Rápido)**
        *   **¿Qué hace?** Utiliza expresiones regulares (`regex`) y patrones predefinidos para identificar rápidamente información de contacto común (ej. formatos de email, números de teléfono, enlaces a LinkedIn).
        *   **Ventaja:** Es muy rápido y eficiente para datos estructurados y predecibles.
        *   **Herramientas:** Principalmente lógica de `regex`.

    *   **Capa 2: Procesamiento del Lenguaje Natural (NLP)**
        *   **¿Qué hace?** Analiza el texto del correo y la firma para entender su significado y contexto. Identifica entidades como Nombres de Personas, Nombres de Organizaciones, Direcciones, etc.
        *   **Ventaja:** Puede extraer información incluso si no sigue un patrón exacto, entendiendo el "sentido" del texto.
        *   **Herramientas:** Modelos de `spaCy` (como `en_core_web_sm` y `es_core_news_sm`).

    *   **Capa 3: Visión por Computadora (OCR)**
        *   **¿Qué hace?** Si la firma de correo es una imagen (en lugar de texto plano), esta capa entra en acción. Utiliza el Reconocimiento Óptico de Caracteres para "leer" el texto dentro de esas imágenes.
        *   **Ventaja:** Permite extraer datos de firmas que están incrustadas como imágenes, una fuente común de información de contacto.
        *   **Herramientas:** `OpenCV` (para procesamiento de imágenes) y `Tesseract` (el motor OCR).

    *   **Capa 4: Refinamiento con Inteligencia Artificial (IA)**
        *   **¿Qué hace?** Una vez que las capas anteriores han extraído información, esta capa utiliza un modelo avanzado de IA (Gemini Flash) para validar semánticamente los datos. Por ejemplo, verifica si un nombre o un cargo parece coherente, o si un número de teléfono tiene el formato correcto.
        *   **Ventaja:** Asegura la integridad y la calidad de los datos finales, corrigiendo posibles errores de las capas anteriores y estructurando la información de forma más fiable.
        *   **Herramientas:** Modelo `Gemini Flash` (a través de su API).

3.  **Salida y Almacenamiento:**
    *   Los contactos extraídos y validados se guardan en un archivo **Excel (`data/contactos.xlsx`)**.
    *   Este archivo se organiza alfabéticamente por institución para facilitar su consulta.
    *   Todos los eventos y operaciones del sistema se registran en **archivos de log (`logs/system.log`)**, con anonimización de datos sensibles para proteger la privacidad.

## 🛠️ Tecnologías Clave Utilizadas

*   **Python 3.9+**: Lenguaje de programación principal.
*   **IMAP**: Protocolo para leer y monitorear correos electrónicos.
*   **spaCy**: Librería de NLP para el análisis semántico de texto.
*   **OpenCV**: Librería de visión por computadora para el procesamiento de imágenes.
*   **Tesseract OCR**: Motor de reconocimiento óptico de caracteres para extraer texto de imágenes.
*   **API de Gemini Flash**: Modelo de IA para validación y refinamiento semántico de los datos.
*   **Pandas**: Librería para el manejo eficiente de datos, especialmente para la interacción con Excel.

## 📦 Componentes del Proyecto (Estructura de Archivos)

Para entender cómo se organiza el código, aquí están los componentes principales y su función:

### 1. **Archivos en la Raíz del Proyecto:**

*   **`main.py`**:
    *   **El "Cerebro"**: Es el archivo principal que se ejecuta para iniciar el servicio. Orquesta todo el flujo de trabajo: conecta con el correo, inicia el monitoreo y lanza el proceso de extracción.
*   **`README.md`**:
    *   **El "Manual"**: Describe el proyecto, sus características, requisitos, cómo instalarlo y cómo usarlo. Es tu primera fuente de información.
*   **`requirements.txt`**:
    *   **La "Lista de Ingredientes"**: Contiene todas las librerías de Python (como `spaCy`, `opencv`, `pandas`) que el proyecto necesita. Se usa para instalar todo automáticamente con `pip`.
*   **`.env.example`**:
    *   **La "Plantilla de Configuración Segura"**: Un archivo de ejemplo que muestra qué variables de entorno necesitas configurar (ej. tu usuario y contraseña de email, claves de la API de Gemini). El archivo real (`.env`) nunca se sube a Git por seguridad.
*   **`list_models.py`**:
    *   **La "Herramienta de Diagnóstico de Modelos"**: Un script utilitario para verificar si los modelos de IA o NLP (como los de spaCy) están instalados y disponibles correctamente.
*   **`test_api.py`**:
    *   **El "Control de Calidad"**: Contiene pruebas para asegurar que las diferentes partes del sistema, especialmente las interacciones con APIs externas (como Gemini), funcionan como se espera.

### 2. **Directorio `app/` (El "Motor Interno"):**

Aquí se encuentra la lógica de negocio modularizada.

*   **`config.py`**:
    *   **El "Panel de Control"**: Lee las variables de entorno de tu archivo `.env` y las pone a disposición del resto de la aplicación. Esto centraliza y protege la configuración.
*   **`email_reader.py`**:
    *   **El "Lector de Correos"**: Se encarga de toda la comunicación con el servidor IMAP: se conecta, monitorea la bandeja de entrada, lee el contenido de los correos y los marca como procesados.
*   **`extractor.py`**:
    *   **El "Analista Inteligente"**: Este es el corazón del sistema de extracción. Aquí residen las lógicas de las capas heurística, NLP, visión (OCR) y refinamiento con IA (Gemini). Recibe el contenido de un correo y devuelve los datos de contacto estructurados.
*   **`excel_manager.py`**:
    *   **El "Gestor de Base de Datos (Excel)"**: Se encarga de interactuar con el archivo `contactos.xlsx`. Lee contactos existentes, añade nuevos y actualiza la información, asegurando que los datos estén bien organizados.
*   **`logger.py`**:
    *   **El "Registrador de Eventos"**: Maneja todos los mensajes de registro del sistema (errores, advertencias, información). Envía estos mensajes al archivo `logs/system.log`, asegurando que la información sensible se anonimice.
*   **`utils.py`**:
    *   **Las "Herramientas Varias"**: Contiene funciones de utilidad pequeñas y reutilizables que no encajan en ningún otro módulo específico (ej. funciones de formateo de texto, validaciones genéricas).

### 3. **Directorio `data/` (El "Almacén de Resultados"):**

*   **`contactos.xlsx`**: El archivo Excel donde se guardan todos los contactos extraídos y validados.

### 4. **Directorio `logs/` (El "Historial de Operaciones"):**

*   **`system.log`**: El archivo donde se registra toda la actividad del servicio, útil para depuración y monitoreo.

## 🚀 Cómo Iniciar el Servicio

1.  **Preparación del Entorno (Termux/Linux):**
    *   Instala Python y dependencias del sistema operativo:
        ```bash
        pkg update && pkg upgrade
        pkg install python tesseract libjpeg-turbo libpng pandas opencv-python
        ```
2.  **Instalar Dependencias de Python:**
    *   Ejecuta: `pip install -r requirements.txt`
    *   Descarga los modelos de spaCy:
        ```bash
        python -m spacy download en_core_web_sm
        python -m spacy download es_core_news_sm
        ```
3.  **Configurar Credenciales:**
    *   Crea un archivo `.env` en la raíz del proyecto (copiando `.env.example`) y rellena tus datos:
        ```env
        EMAIL_USER="tu_usuario@gmail.com"
        EMAIL_PASS="tu_contraseña_de_aplicación"
        GEMINI_API_KEY_1="tu_llave_api_principal"
        ```
        *(Recuerda que `EMAIL_PASS` debe ser una "contraseña de aplicación" generada por tu proveedor de correo, no tu contraseña principal, para mayor seguridad).*
4.  **Iniciar el Servicio:**
    *   Ejecuta: `python main.py`
    *   El servicio se iniciará y comenzará a monitorear tu bandeja de entrada en tiempo real.

## ✅ Valor y Beneficios del Proyecto

*   **Automatización:** Elimina la necesidad de entrada manual de datos de contacto de correos electrónicos.
*   **Eficiencia:** Procesa grandes volúmenes de correos de forma rápida y continua.
*   **Precisión:** Utiliza IA y NLP para asegurar la alta calidad y coherencia de los datos extraídos.
*   **Datos Estructurados:** Transforma información desorganizada en un formato útil (Excel).
*   **Base de Datos Actualizada:** Mantiene una base de datos de contactos dinámica y centralizada.
*   **Auditoría:** Los logs permiten rastrear la actividad y solucionar problemas.
